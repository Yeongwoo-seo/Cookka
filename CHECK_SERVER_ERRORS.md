# 서버 오류 확인 가이드

## 문제
500 Internal Server Error가 계속 발생합니다.

## 해결 방법

### 1. 개발 서버 재시작
변경사항을 적용하기 위해 개발 서버를 재시작하세요:

```powershell
# 현재 실행 중인 서버 중지 (Ctrl+C)
# 그 다음 다시 시작
npm.cmd run dev
```

### 2. 서버 로그 확인
개발 서버 터미널에서 다음 정보를 확인하세요:

1. **오류 메시지**: 정확한 오류 메시지
2. **오류 위치**: 어떤 파일에서 오류가 발생했는지
3. **스택 트레이스**: 오류가 발생한 함수와 라인 번호

### 3. 가능한 원인

#### 원인 1: Firebase SDK 모듈 레벨 Import
`lib/firebase.ts`가 모듈 레벨에서 Firebase SDK를 import하고 있어서, 서버 사이드에서 실행될 때 문제가 발생할 수 있습니다.

**해결책**: Firebase SDK를 동적으로 import하도록 변경 (복잡할 수 있음)

#### 원인 2: 다른 모듈에서 Firebase Import
다른 파일에서 Firebase를 모듈 레벨에서 import하고 있을 수 있습니다.

**확인 방법**: 
```bash
grep -r "from.*firebase" --include="*.ts" --include="*.tsx" .
```

#### 원인 3: 환경 변수 누락
Firebase 환경 변수가 설정되지 않았을 수 있습니다.

**확인 방법**: `.env.local` 파일 확인

### 4. 디버깅 단계

1. **개발 서버 재시작**
2. **브라우저에서 접속**: `http://localhost:8080`
3. **서버 로그 확인**: 터미널에서 오류 메시지 확인
4. **오류 메시지 공유**: 정확한 오류 메시지를 공유해 주세요

## 다음 단계

서버 로그의 정확한 오류 메시지를 공유해 주시면, 더 구체적인 해결책을 제시할 수 있습니다.
